Evaluando portales de datos

Un portal de datos debe ofrecer datos de calidad, confiables y que sean fácilmente reutilizables.

Para poder evaluar si lo logran, definimos los siguientes criterios, basado en el trabajo del Open Data Index y de Fundación Conocimiento Abierto (Licencia CC-BY-SA 4.0)

Evaluando via API

Los portales de datos deben publicar una API (interfaz programática) que permita acceso automatizado a sus datos y metadatos. Mejor aún si implementan el standard DCAT (data.json).

Esto permite evaluar cada uno de los datasets.

datapackages y goodtables

  1. No los actualizan
  2. No son fáciles de reutilizar
  3. No tienen licencia o responsable
  4. No hay forma de reportar errores

Responsable

Los metadatos del conjunto de datos deben expresar quién es el responsable del mismo, esto incrementa la confiabilidad y facilita la retroalimentación.

La dirección de correo electrónico del responsable debe estar publicada.

Frecuencia de actualización declarada

Una expectativa para la actualización de los datos debe estar definida en metadatos del conjunto de datos, esto aumenta la confiabilidad y permite planificar análisis periódicos.

La frecuencia de actualización no debe decir "a demanda", "eventual" o ser inespecífica en algún otro sentido.

Frecuencia de actualización efectiva

El tiempo transcurrido desde la última fecha de actualización no debe ser superior a la frecuencia de actualización declarado. Los datos actualizados permiten realizar análisis con información más reciente.

Licencia

Los términos de la licencia con la que se comparte un dataset deben estar expresados en los metadatos de cada conjunto de datos. La licencia debe habilitar el uso para cualquier fin y requiriendo, como máximo, la atribución. Esto evita dudas sobre la necesidad de pedir autorización para la reutilización.

Licencias: odc-pddl, odc-odbl, odc-by, cc-zero, cc-by, cc-by-sa, gfdl

Formato

El conjunto de datos debe contener al menos un recurso cuya extensión de archivo corresponda con la de un formato abierto. Los formatos abiertos son aquellos cuya interpretación o reproducción no está limitada por patentes o regalías. Esto elimina barreras para la reutilización.

Fromatos: csv, json, xml, geojson, kml

Validez

Los recursos de cada conjunto de datos que puedan ser validados serán evaluados en su adherencia a los requisitos del formato, lo que facilita su reutilización.
